
## notes

现在的问题是，虽然在 `BitExpr` $\to$ `ANFPolynomial` 的时候应用 `ReadBitExpr` 的替换和展开是非常方便的，但是此时的展开并未发生在 ANF 内，缺少环境单项式/多项式信息，不利于在数值映射时展开策略和后续分析；
- 我们通过当前的单项式次数来确定是否展开。只有 2 次以上的单项式才有展开的必要；线性项可以直接保留（甚至直接优化为单个变量）
- 对于决定展开的变量，需要记录“单项式次数的序关系”，以便后续进行单项式优化。具体而言，若变量 $c$ 可替换为 $a_1a_2+\cdots$，那么就有 $\mathtt{DEG}(c)\leq\mathtt{DEG}(a_1a_2)$。进行单项式优化时，若遇到单项式 $A$ 且 $a_1a_2\prec A$，那么久可以将 $\mathtt{DEG}(A)$ 记作 $\mathtt{DEG}(c)\cdot\mathtt{DEG}(A/a_1a_2)$。
